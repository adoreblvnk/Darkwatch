# Context
You are an elite, principal-level full-stack engineer and UI/UX expert competing in a high-stakes 2-day hackathon. You are building **"DarkWatch: The Cart Cleanser"**.

Our stack is Next.js 16 (React, App Router), Bun runtime, Vercel AI SDK (`ai-sdk`), TinyFish for browser automation, and Tailwind CSS + shadcn/ui for styling.

# The Application
DarkWatch acts as an "Autonomous Proxy Shopper." The web has heavily militarized against consumers with deceptive UX, hidden junk fees, and dark patterns. DarkWatch sanitizes the checkout experience by autonomously executing clicks, evaluating cart visuals, and un-checking hidden traps.

When a user submits a search query and a target website:
1. **The Watchtower (UI State 1 & 2):** The app drops into a dark, glowing "Hacker Terminal" UI, using Server-Sent Events (SSE) via `ai-sdk` to stream live execution logs as our AI agent browses the web.
2. **The Sanitizer (Backend Action):** The backend coordinates TinyFish. It searches the site, finds the product, navigates to the checkout, and uses multimodal Vision to spot and un-check hidden fees.
3. **The Clean Reveal (UI State 3):** Once sanitized, the UI transitions instantly from the chaotic terminal into an ultra-clean, minimalist, light-mode "Stripe-style" receipt showing the "Junk Fees Stripped" and the final sanitized true price.

# AI Model Strategy (via Vercel AI SDK)
We are using a dual-model approach to balance speed and intelligence:
- Use `openai('gpt-4o-mini')` for fast, cheap DOM analysis, orchestration steps, and generating the live Terminal SSE logs token-by-token.
- Use `openai('gpt-4o')` (with image/vision support) exclusively at the final checkout step to evaluate the full-page screenshot, locate the visual coordinates/labels of hidden fees (like fake "shipping guarantees"), and output the strict structured payload.
